## type-design-analyzer

### Findings (type-design-analyzer)

- **HIGH** `GuardDecision` (`crates/agent-guard-core/src/decision.rs`) â€” All three variants are public struct/tuple variants with no `#[non_exhaustive]` guard, so an external crate can construct `Deny` with an empty `message` or `AskUser` with an empty `message` field, bypassing the smart constructors that are the only intended entry points.
  ```rust
  let d = GuardDecision::Deny {
      reason: DecisionReason {
          code: DecisionCode::DeniedByRule,
          message: String::new(),
          details: None,
          matched_rule: None,
      },
  };
  ```
  remediation: Mark `GuardDecision` and `RuntimeDecision` variants `#[non_exhaustive]` and make `DecisionReason` fields `pub(crate)`, forcing all construction through the provided smart constructors.

- **HIGH** `GuardDecision` / `DecisionReason` (`crates/agent-guard-core/src/decision.rs`) â€” `serde::Deserialize` is derived with no custom validation, so a payload like `{"decision":"deny","code":"DENIED_BY_RULE","message":""}` produces a `Deny` with empty `message`, and an `ask_user` payload can produce empty prompt and reason; neither is reachable via smart constructors.
  ```rust
  let d: GuardDecision = serde_json::from_str(
      r#"{"decision":"deny","code":"DENIED_BY_RULE","message":""}"#
  ).unwrap();
  ```
  remediation: Implement a custom `Deserialize` for `GuardDecision` / `DecisionReason` that rejects empty `message` strings, or wrap `message` in a `NonEmptyString` newtype.

- **MEDIUM** `DecisionReason` (`crates/agent-guard-core/src/decision.rs`) â€” All four fields are `pub`, letting any caller mutate `message` to empty or swap `code` after construction, undermining audit-trail immutability of an emitted `Deny`.
  ```rust
  let mut d = GuardDecision::deny(DecisionCode::DeniedByRule, "blocked by policy");
  if let GuardDecision::Deny { ref mut reason } = d {
      reason.message = String::new();
      reason.code = DecisionCode::InternalError;
  }
  ```
  remediation: Change `DecisionReason` fields to `pub(crate)` with read-only accessors; existing builder methods already return `Self` by value, so external mutation is unnecessary.
